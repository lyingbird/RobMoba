## Project Overview

RobMoba is a MOBA (Multiplayer Online Battle Arena) game built for Roblox Studio. The project implements core MOBA mechanics including champions with unique skill sets, items, runes, leveling, enemy AI, and a full client-server architecture.

## Tech Stack

- **Language:** Lua / Luau (Roblox dialect)
- **Platform:** Roblox Studio
- **Architecture:** Client-server (Roblox replication model)
- **Project structure:** Rojo-compatible `src/` layout

## Project Structure

```
src/
  ReplicatedStorage/       -- Shared configs (skills, items, runes, levels)
  ServerScriptService/     -- Server logic (skill manager, enemy manager)
    ServerModules/         -- Core modules (BaseSkill, EnemyClass, StatsManager, InventoryManager)
      Skills/              -- Per-skill implementations (Skill_1001 .. Skill_1015)
  ServerStorage/           -- Plugins (SkillEditorPlugin)
  StarterPlayer/
    StarterPlayerScripts/  -- Client scripts (UI, input, camera, effects)
      Modules/             -- Client modules (CameraManager, CooldownManager, etc.)
      UIComponents/        -- UI elements (HUD, Backpack, DragDrop)
```

## Key Systems

- **Skill system:** Data-driven via SkillConfig. Each skill has a BaseSkill superclass and a specific Skill_XXXX module. Supports cooldowns, runes, recast mechanics, and cinematic ultimates.
- **Champion roster:** Lux, Angela, HouYi, LianPo -- each with Q/W/R abilities.
- **Stats & leveling:** LevelConfig defines XP curves and per-level stat growth (HP, MP, ATK, DEF, etc.).
- **Items & runes:** ItemConfig and RuneConfig provide equipment and skill augmentation data.
- **Enemy AI:** EnemyClass with EnemyManager for spawning and lifecycle.
- **Client UI:** Modular UI system with HUD, backpack, drag-drop, cooldown display, and overhead health bars.

## Coding Conventions

- Server scripts use `.server.lua` suffix; client scripts use `.client.lua`.
- Shared modules live in ReplicatedStorage; server-only modules in ServerScriptService/ServerModules.
- Chinese UI strings are used for player-facing text (UIName fields).
- Config tables use numeric IDs as keys (e.g., `[1001]` for Fireball).
