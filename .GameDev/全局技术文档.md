# RobMoba 全局技术文档

> 最后更新: 2026-03-13
> 版本: v1.1

## 技术架构概览

```
Client-Server Architecture (Roblox Replication Model)
├── Server (ServerScriptService + ServerModules)
│   ├── LobbyManager ★ REQ-004 — 大厅+匹配核心
│   ├── DuelManager ★ REQ-004 — 对决生命周期
│   ├── MatchSystem — 战斗追踪 (被DuelManager调用)
│   ├── GameManager ⛔ 废弃 (do return end)
│   ├── PlayerSkillManager — 技能装备/释放/冷却
│   ├── AutoAttackManager — 普攻伤害/动画
│   ├── StatsManager — 属性计算 (Attribute桥)
│   ├── InventoryManager — 背包/装备
│   ├── RemoteEventInit — 16个RemoteEvent
│   └── EnemyManager — NPC敌人
├── Shared (ReplicatedStorage)
│   └── Config表: SkillConfig, HeroConfig, ItemConfig, LevelConfig, RuneConfig
└── Client (StarterPlayer)
    ├── Client.client — 入口，大厅模式
    ├── InputManager — LOL风格输入
    ├── UI_HUD — HUD/技能栏
    ├── UI_HeroSelect — 左下角英雄面板 (REQ-004)
    ├── UI_MatchButton — 右下角匹配按钮 (REQ-004)
    └── CameraManager, MovementManager, HeroAnimator, etc.
```

## 核心系统

### 游戏流程系统 (REQ-002 → REQ-004 替代)
- **GameManager.server.lua**: ⛔ 已废弃 (`do return end`)，原5状态状态机不再使用
- **LobbyManager.server.lua** ★ REQ-004: 大厅核心调度，管理玩家状态 (LOBBY/MATCHING/DUELING)、匹配队列、英雄切换
- **DuelManager.server.lua** ★ REQ-004: 对决生命周期 — 阵营分配(Teams)、传送、倒计时、调用MatchSystem战斗追踪、结算回送大厅
- **通信协议**: MatchmakingEvent (双向)、DuelEvent (S→C)、HeroSwapEvent (C→S)
- **玩家状态**: `LOBBY`(自由活动) → `MATCHING`(排队中) → `DUELING`(对决中)
- **技术文档**: [查看详情](.GameDev/REQ-004/03_技术设计.md)

### 战斗系统
- **MatchSystem**: 击杀计数、死亡重生 (5s)、胜负判定 (KILLS_TO_WIN=3)
- **CombatUtils**: 统一敌我判断 (Team-based)，大厅无Team时NPC仍可攻击
- **AutoAttackManager**: 伤害公式 ATK*(100/(100+DEF))

### 技能系统
- **数据驱动**: SkillConfig → BaseSkill → Skill_XXXX (15个)
- **通信**: CastSkillEvent, EquipSkillEvent, SyncCooldownEvent
