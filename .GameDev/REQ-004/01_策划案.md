# REQ-004: 自由大厅 + 匹配区域 + 1v1 对决

> 版本: v1.0
> 状态: 策划中
> 类型: FEATURE (XL)
> 参考: The Strongest Battlegrounds, Blade Ball, Untitled Boxing Game

---

## 1. 设计目标

**核心理念**: 让玩家进入游戏即可自由活动，**零等待、零门槛**。PvP 对战是可选的、主动触发的，而非强制的。

**参考模型**: Roblox 格斗/战斗类游戏的标准模式
- 玩家进入 → 自由漫游大厅 → 选英雄打木桩练技能
- 想 PvP → 走进匹配区域 → 自动寻找对手 → 进入 1v1 对决
- 对决结束 → 回到大厅继续自由活动

**与旧流程对比**:

| 项目 | 旧流程 (GameManager) | 新流程 (大厅模式) |
|------|---------------------|------------------|
| 进入体验 | 等待→选英雄→加载→战斗 | 直接进入大厅，自由活动 |
| 单人体验 | 什么都做不了/黑屏 | 选英雄打木桩练技能 |
| 多人体验 | 强制 PvP | 自愿进入匹配区域 |
| 英雄选择 | 倒计时强制选 | 随时在大厅切换 |

## 2. 核心模块拆分

### M1: 大厅系统 (LobbySystem)

**职责**: 管理玩家在大厅的状态和行为

**功能**:
- 玩家进入游戏后**立即生成角色**在出生点
- `CharacterAutoLoads = true`（默认行为，不修改）
- 大厅中玩家**不能互相攻击**（非 PvP 区域）
- 大厅中有**木桩/训练假人**可供练习
- 玩家在大厅可以**自由移动、释放技能打木桩**

**状态**:
```
LOBBY    — 在大厅自由活动（默认状态）
MATCHING — 在匹配区域等待对手
DUELING  — 正在 1v1 对决中
```

**王者荣耀参考**: 类似"训练营"模式 + 大厅社交区域的结合

### M2: 英雄选择系统 (改造 HeroSelect)

**职责**: 在大厅中随时切换英雄

**功能**:
- 进入大厅后，**左下角**显示英雄选择小面板（显眼但不占空间）
- 小面板样式：当前英雄头像 + 英雄名 + "切换"按钮
- 点击"切换"按钮或首次进入 → 展开完整英雄选择列表
- 选择英雄后：角色立即装备对应技能，小面板更新为新英雄头像
- 在大厅中可以**随时点击左下角面板重新选择英雄**
- 切换英雄时：重新装备技能、重新初始化 HeroAnimator
- **对决中锁定**：进入 MATCHING/DUELING 状态后，英雄选择面板灰化不可操作

**UI 布局**:
```
┌──────────────────────────────────────────┐
│                                          │
│                 游戏画面                  │
│                                          │
│                                          │
│                                          │
│ ┌──────────┐              ┌────────────┐ │
│ │ [头像]   │              │  ⚔️ 匹配   │ │
│ │ 后羿     │   技能栏     │  开始匹配   │ │
│ │ [切换▼]  │  Q W R D F  │            │ │
│ └──────────┘              └────────────┘ │
└──────────────────────────────────────────┘
   左下角                       右下角
```

**与旧系统区别**: 不再是全屏弹窗倒计时强制选择，而是左下角常驻小面板，随点随选

### M3: 匹配区域 + 匹配按钮 (MatchmakingZone + UI)

**职责**: 双入口触发匹配 — 物理区域 + UI 按钮

**入口1: 物理匹配区域**
- 在出生点旁边放置一个**视觉显著的匹配区域**（发光圆台/粒子效果）
- 玩家**走进区域** → 进入 MATCHING 状态
- 玩家**走出区域** → 取消匹配，回到 LOBBY 状态
- 区域上方显示**当前匹配中的玩家数**和等待提示
- 匹配区域有明确的视觉标识：
  - 地面发光圆环（蓝色/金色交替脉冲）
  - 上方悬浮文字 "⚔️ PvP 匹配" + 当前人数
  - 进入时屏幕边缘闪烁提示 "正在匹配..."

**入口2: 右下角匹配按钮（⭐ 用户确认）**
- **右下角**常驻匹配按钮："⚔️ 开始匹配"
- 按下 → 进入 MATCHING 状态（无需走到物理区域）
- 匹配中按钮变为 "❌ 取消匹配"（红色），再按取消
- 匹配中按钮显示实时等待信息："匹配中... 1/2"
- 匹配成功 → 按钮不可点击，显示 "对手已找到！"
- **对决中隐藏**匹配按钮

**两个入口效果完全一致**：都调用同一个服务端匹配接口，状态同步。走进物理区域时右下角按钮也同步变为"取消匹配"状态。

**The Strongest Battlegrounds 参考**: TSB 中玩家直接在开放世界攻击他人。我们采用更友好的"匹配区域 + 匹配按钮"双入口模式，方便不同习惯的玩家。

### M4: 匹配管理器 (MatchmakingManager)

**职责**: 服务端匹配逻辑

**功能**:
- 维护匹配队列 `matchQueue = { player1, player2, ... }`
- 当队列中有 **≥ 2 个玩家** → 取出前两个，创建对决
- 匹配在**当前服务器内**进行（不跨服）
- 匹配成功 → 广播双方 → 进入对决流程

**匹配流程**:
```
玩家A走进匹配区域 → 加入matchQueue → 显示"等待对手..."
玩家B走进匹配区域 → 加入matchQueue → 检测队列≥2
    → 取出A和B → 通知双方"匹配成功！" → 3秒倒计时 → 传送到对决场地
```

**单人时的处理**: 匹配区域显示 "当前排队: 1/2 — 等待对手加入服务器..."。玩家可以随时走出取消。

### M5: 对决系统 (DuelSystem)

**职责**: 管理 1v1 对决的完整生命周期

**功能**:
- 匹配成功后，**传送双方到对决场地**（独立区域或在大厅旁边的竞技场）
- 3 秒倒计时 → 开战
- **复用现有 MatchSystem 的战斗逻辑**: 阵营分配、击杀判定、3杀胜利、死亡重生
- 对决结束 → 显示结果 → 5 秒后传送回大厅
- 对决中**其他玩家不受影响**，继续在大厅活动

**对决场地选项**:
- 方案 A: 大厅旁边的围栏竞技场（简单，其他玩家可观战）
- 方案 B: 独立隔离区域（防干扰但无观战）
- **推荐方案 A** — Roblox 社交性更强，观战有趣

## 3. 详细交互流程

### 3.1 玩家首次进入

```
进入服务器
    │
    ├── 角色自动生成在出生点
    ├── 基础UI初始化（MOBA_HUD + 头顶血条 + 属性面板）
    ├── 弹出英雄选择界面
    │   ├── 选择英雄 → 装备技能 → 关闭选择界面
    │   └── 不选（保持默认/上次英雄）
    │
    └── 进入 LOBBY 状态 → 自由活动
        ├── 打木桩练技能
        ├── 在大厅跑动探索
        └── 走进匹配区域 → 进入匹配流程
```

### 3.2 匹配流程

```
走进匹配区域
    │
    ├── 服务端: 加入 matchQueue
    ├── 客户端: 显示匹配UI（"正在匹配... 1/2"）
    ├── 头顶显示"🔍 匹配中"标识
    │
    ├── 等待中:
    │   ├── 可以走出区域取消匹配
    │   └── 匹配区域上方实时显示人数
    │
    └── 找到对手:
        ├── 双方屏幕弹出 "对手已找到！" + 对手信息
        ├── 3秒倒计时（"3...2...1... FIGHT!"）
        └── 传送到竞技场
```

### 3.3 对决流程

```
传送到竞技场
    │
    ├── 分配阵营（红/蓝）
    ├── 双方出生在竞技场两端
    ├── 倒计时结束 → 解锁操作
    │
    ├── 对战中:
    │   ├── 使用现有战斗系统（普攻+技能）
    │   ├── 死亡 → 5秒重生 → 继续
    │   ├── 击杀计数显示（0 VS 0）
    │   └── 先达3杀 → 胜利
    │
    └── 结算:
        ├── 弹出结算界面（胜利/失败 + 数据）
        ├── 5秒后自动传送回大厅出生点
        └── 状态重置为 LOBBY
```

## 4. 服务端-客户端协议

### 新增 RemoteEvent

| 事件名 | 方向 | 用途 |
|--------|------|------|
| `MatchmakingEvent` | 双向 | 客户端→服务端: 加入/离开匹配队列; 服务端→客户端: 匹配状态更新 |
| `DuelEvent` | S→C | 对决开始/结束/倒计时/结算 |
| `HeroSwapEvent` | C→S | 在大厅中切换英雄 |

### 复用现有 RemoteEvent

| 事件名 | 复用方式 |
|--------|---------|
| `CastSkillEvent` | 技能释放（不变） |
| `EquipSkillEvent` | 技能装备（切换英雄时重新装备） |
| `AttackTargetEvent` | 普攻（不变） |
| `MatchStateEvent` | 对决中击杀计数（复用） |
| `DeathTimerEvent` | 对决中死亡倒计时（复用） |

## 6. 验收标准

### AC-001 ～ AC-005: 大厅基础
- **AC-001**: 玩家进入游戏后 ≤ 3 秒内角色出现在出生点，可自由移动
- **AC-002**: 英雄选择界面在进入后自动弹出，可选择 5 个英雄中的任意一个
- **AC-003**: 选择英雄后，Q/W/R 技能自动装备到 HUD 技能栏并显示图标
- **AC-004**: 大厅中可对木桩释放技能，造成伤害（木桩血条变化）
- **AC-005**: 大厅中多个玩家**不能互相攻击**（CombatUtils 区分大厅/对决状态）

### AC-006 ～ AC-010: 匹配区域
- **AC-006**: 出生点旁存在视觉显著的匹配区域（发光效果+悬浮文字）
- **AC-007**: 走进匹配区域后，屏幕显示"正在匹配... X/2"
- **AC-008**: 走出匹配区域后，匹配状态取消，UI 消失
- **AC-009**: 匹配区域上方实时显示当前排队人数
- **AC-010**: 只有已选择英雄的玩家才能进入匹配（未选英雄弹出提示）

### AC-011 ～ AC-015: 匹配与对决
- **AC-011**: 两个玩家同时在匹配区域时，≤ 2 秒内匹配成功
- **AC-012**: 匹配成功后双方屏幕显示对手信息 + 3 秒倒计时
- **AC-013**: 倒计时结束后传送到竞技场，双方在竞技场两端出生
- **AC-014**: 对决使用现有战斗系统，先达 3 杀获胜
- **AC-015**: 对决结束后显示结算界面，5 秒后自动传送回大厅

### AC-016 ～ AC-018: 边界与异常
- **AC-016**: 对决中一方掉线 → 另一方自动获胜 → 回到大厅
- **AC-017**: 匹配中掉线 → 自动从匹配队列移除
- **AC-018**: 服务器只有 1 个玩家时，匹配区域显示"等待其他玩家加入..."

## 7. 数值参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 匹配区域半径 | 15 studs | 匹配区域触发范围 |
| 匹配倒计时 | 3 秒 | 匹配成功后到传送的等待时间 |
| 对决击杀数 | 3 | 先达 3 杀获胜（与现有一致） |
| 重生时间 | 5 秒 | 对决中死亡重生时间 |
| 结算持续 | 5 秒 | 结算界面显示时间 |
| 竞技场尺寸 | 100 × 100 studs | 对决场地大小 |
| 竞技场出生距离 | 80 studs | 双方出生点间距 |
