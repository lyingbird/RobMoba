### 3.2 玩家状态流转 (服务端权威)

```
玩家加入服务器
    │
    ▼
  LOBBY ◄──────────────────────────┐
    │                               │
    ├── 按匹配按钮/走进区域         │
    ▼                               │
  MATCHING                          │
    │                               │
    ├── 按取消/走出区域 → LOBBY     │
    │                               │
    ├── 队列≥2 → 匹配成功          │
    ▼                               │
  DUELING ──── 对决结束 ────────────┘
```

### 3.3 关键时序：匹配 → 对决 → 回大厅

```
Client A          LobbyManager        DuelManager        Client B
   │                   │                   │                  │
   │──MatchmakingEvent─▶                   │                  │
   │  {action:"join"}  │                   │                  │
   │                   │◄─MatchmakingEvent─│──────────────────│
   │                   │  {action:"join"}  │                  │
   │                   │                   │                  │
   │                   │── 检测队列≥2 ────▶│                  │
   │                   │   创建对决        │                  │
   │                   │                   │                  │
   │◄──DuelEvent───────│───────────────────│──DuelEvent──────▶│
   │ {type:"matched"}  │                   │ {type:"matched"} │
   │                   │                   │                  │
   │   (3秒倒计时)     │                   │   (3秒倒计时)    │
   │                   │                   │                  │
   │◄──DuelEvent───────│───────────────────│──DuelEvent──────▶│
   │ {type:"start"}    │  传送+分配阵营    │ {type:"start"}   │
   │                   │  StartBattle()    │                  │
   │                   │                   │                  │
   │     ... 战斗中 (MatchSystem 处理) ... │                  │
   │                   │                   │                  │
   │◄──DuelEvent───────│───────────────────│──DuelEvent──────▶│
   │ {type:"result"}   │  EndBattle()      │ {type:"result"}  │
   │                   │  传送回大厅       │                  │
   │                   │  状态→LOBBY       │                  │
```

## 4. 接口设计

### 4.1 RemoteEvent 协议

**MatchmakingEvent (双向)**:
```lua
-- C→S: 加入匹配
MatchmakingEvent:FireServer({ action = "join" })
-- C→S: 离开匹配
MatchmakingEvent:FireServer({ action = "leave" })
-- S→C: 状态更新
MatchmakingEvent:FireClient(player, {
    status = "queued",     -- "queued" | "cancelled" | "matched"
    queueSize = 1,         -- 当前队列人数
    opponent = nil,         -- 匹配成功时: { name, heroId }
})
```

**DuelEvent (S→C)**:
```lua
DuelEvent:FireClient(player, {
    type = "matched",       -- "matched" | "countdown" | "start" | "result"
    -- matched: opponent = { name, heroId }
    -- countdown: seconds = 3
    -- start: team = "RedTeam"/"BlueTeam", arenaCenter = Vector3
    -- result: winner = "RedTeam", stats = { ... }
})
```

**HeroSwapEvent (C→S)**:
```lua
HeroSwapEvent:FireServer({ heroId = "HouYi" })
-- S→C 确认:
HeroSwapEvent:FireClient(player, {
    success = true,
    heroId = "HouYi",
    message = nil,  -- 失败时: "正在匹配中，无法切换"
})
```

### 4.2 LobbyManager 服务端数据结构

```lua
local playerStates = {}  -- { [Player] = "LOBBY" | "MATCHING" | "DUELING" }
local playerHeroes = {}  -- { [Player] = "HouYi" | nil }

```

### 4.3 CombatUtils.isEnemy 改造

```lua
-- 现有逻辑（第72-78行）:
-- 没有 Team 分配时，return false（不允许攻击）

-- 改造后:
-- 规则1: NPC 永远是敌方（不变）
-- 规则2: 有 Team → 不同 Team = 敌方（不变）
-- 规则3: 无 Team 且双方都是玩家 → return false（大厅不互殴）
-- 无需新增字段，利用现有 Team 机制：大厅无Team→不能打玩家；对决有Team→可以打
```

## 6. 技术风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| MatchSystem.CharacterAutoLoads=false 影响大厅角色加载 | 高 | 确定 | T1 任务优先移除此行 |
| 对决传送后角色位置/状态不正确 | 中 | 中 | LoadCharacter 后用 CFrame 强制设置 |
| 匹配区域 Touched 事件频繁触发 | 低 | 中 | 防抖: 0.5秒内不重复处理 |
| 对决中一方掉线，另一方卡在竞技场 | 高 | 低 | PlayerRemoving 强制结束对决 |
